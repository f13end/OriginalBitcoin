> Dear marco,

> that is fine for me, how do we do that?

This list is ran with the mailman software, as you can see by reading
the list headers in every message coming through the list. Mailman
_should_ have somewhere a one-time setting to just discard, forever,
all messages not coming from list members. This is the easiest option,
fire and forget.

If I'm wrong on this, mailman administration interface DOES have for
sure a section to "tend to pending administrative request" and there
should be an option to discard all pending messages with one click. I
used it in the past when I administered lists. In this second case,
you still have to delete those mesages to avoid filling the server,
but is a one-minute task every week or two, depending on how much spam
the list gets as an average.

But I'm pretty sure there must be some once-and-for-all configuration
solution to do this, mailman is so widely used that this problem must
have happened hundreds of times before to other people.

In any case, it's just a matter of checking in the documentation. The
*real* problem is PR/social, so to speak: is it bad if, starting
monday, messages to the lists from unsubscribed people silently go to
the trash bin? (note, personally, the answer is "no", but I'm not the
owner/moderator of course).
