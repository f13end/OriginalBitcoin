On Tuesday, December 7, 2010 12:35:10 PM UTC-6, Dante-Gabryell Monson wrote:

I like the way Michael phrased it : "clarity of agreements",
"standards and protocols by which we might work together"

Personally, I imagine such protocols, based on a semantic web vocabulary,
which can be combined with other existing social networking vocabularies,
such as FOAF
http://en.wikipedia.org/wiki/FOAF_(software)<http://en.wikipedia.org/wiki/FOAF_%28software%29>

<http://en.wikipedia.org/wiki/FOAF_%28software%29>Perhaps some of us are
more accustomed to semantic web terms and ontologies.

Perhaps we can start in a more simple way, by describing such kind of
"protocols" through some card game, which then could be translated into
"programmed" semantic web ontologies for use via the internet ?

Such "game" then being used to make suggestions for networked
interdependencies, which can be queried, and can be approoved of through
contracts which enable the transactions.
Conditions which can use any type of currency. ( private central bank money,
hours, or apples )

To return to a more specific definition for a "Transaction Contract"
vocabulary / agreement protocol,
I could imagine the basic objects to be as simple as
- "User/Agent" ( any person or organization in the system )
- "Project" ( specifically defined in time and space / virtual space )
- "Request"
- "Support"
